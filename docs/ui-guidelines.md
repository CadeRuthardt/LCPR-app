

# UI Guidelines

# Le Chateau Pet Resort Mobile App

Status: Living Document

---

# Purpose

This document defines how the interface should look, feel, and behave.

Its purpose is to ensure every screen feels like it belongs to the same luxury hospitality product.

If a UI decision conflicts with these guidelines, these guidelines take precedence.

---

# Core Philosophy

The interface should feel:

- Luxury
- Calm
- Spacious
- Intentional
- Human
- Premium

Never design screens that resemble kennel software, ERP systems, or enterprise dashboards.

---

# Visual Inspiration

Draw inspiration from:

- Apple
- Airbnb
- Four Seasons
- Aman Resorts
- Mercedes-Benz

Avoid copying their designs directly. Instead, emulate their attention to detail, restraint, and polish.

---

# Color System

Use only the centralized design tokens.

Primary
- Black Cherry (#711012)
- Goldenrod (#D5A727)
- Ivory (#FAFAF0)

Supporting
- Rich Mahogany (#360808)
- Olive Bark (#795E16)
- Graphite (#292929)
- Onyx (#111111)

Color should communicate hierarchy and action—not decoration.

---

# Typography

Typography should create hierarchy before color.

Use:
- Large, confident page titles
- Comfortable line spacing
- Short paragraphs
- Consistent heading scales

Avoid excessive font weights or overly condensed layouts.

---

# Spacing

Whitespace is part of the brand.

Guidelines:

- Favor generous margins.
- Separate content into distinct sections.
- Avoid crowding cards together.
- Let photography breathe.

If a screen feels cramped, remove content before reducing spacing.

---

# Cards

Cards are the primary presentation surface.

Cards should:

- Have rounded corners
- Use subtle elevation
- Clearly separate information
- Be tappable when interactive

Do not overload a single card with too much information.

---

# Buttons

Buttons should be obvious and purposeful.

Priority:

1. Primary Button
2. Secondary Button
3. Text Action

Never place multiple competing primary buttons on the same screen.

---

# Imagery

Photography is a core part of the experience.

Use:

- Real Le Chateau photography
- Happy pets
- Beautiful accommodations
- Genuine staff interactions
- Natural lighting

Avoid generic stock photos whenever possible.

---

# Icons

Icons should support text—not replace it.

Requirements:

- Simple
- Consistent
- Recognizable

Avoid decorative icon usage.

---

# Lists

Lists should feel light.

Each row should have:

- Clear title
- Supporting information
- Optional status
- Chevron when navigable

Avoid dense tables.

---

# Forms

Reservation requests should feel conversational.

Instead of presenting one long form:

- Break requests into steps.
- Explain why information is requested.
- Use progress indicators.
- Validate early.

The guest should feel guided—not interrogated.

---

# Empty States

Every empty state should encourage the next action.

Examples:

No Upcoming Reservation
→ Request a Reservation

No Photos
→ Your pet's memories will appear here after their next visit.

No Active Stay
→ Explore the Resort

---

# Loading States

Never show blank screens.

Use:

- Skeleton loaders
- Progressive loading
- Fade transitions

The interface should always feel responsive.

---

# Motion

Motion should be subtle.

Use:

- Fade
- Slide
- Scale

Animation should support understanding—not entertain.

---

# Tone of Voice

Every word should reinforce hospitality.

Good:

- Welcome Back
- Preparing for Your Stay
- Explore the Resort
- Your Pet's Vacation

Avoid:

- Reservation Type
- Boarding Unit
- Kennel Assignment
- Internal operational terminology

---

# Responsive Principles

Design for:

- iPhone first
- Android second
- Tablet compatibility later

Components should adapt gracefully across screen sizes.

---

# Accessibility

Every screen should:

- Support Dynamic Type where practical
- Meet WCAG contrast guidance
- Provide descriptive labels
- Use accessible touch targets
- Work with screen readers

Accessibility is part of a premium experience.

---

# Final Design Checklist

Before approving any screen, confirm:

- Does it feel like a luxury hospitality brand?
- Is the primary action obvious?
- Is the layout calm and uncluttered?
- Is there unnecessary information that can be removed?
- Are components reused instead of reinvented?
- Does it align with the design system?
- Would this feel at home in a premium hotel app?

If the answer to any question is "no," iterate before implementation.
